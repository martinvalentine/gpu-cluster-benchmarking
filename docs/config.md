# Configuration Reference

## configs/models.yaml

Single source of truth for model downloads, endpoint routing, server startup, and benchmark settings.

### Cluster Configuration

Adjust the `cluster` section to match your pod's GPU count and memory:

```yaml
cluster:
  gpu_count: 0              # 0 = auto-detect from nvidia-smi

  vllm:
    tp: 1                   # Default tensor parallel size
    gpu_mem_util: "0.87"    # GPU memory utilization (0.0-1.0)
    max_model_len: 4096     # Max context length
    max_num_seqs: 64        # Max concurrent sequences
    max_batched_tokens: 8192
    quant: none             # Default quantization: awq, gptq, none
    swap_space: 4           # GB swap for KV cache

  llamacpp:
    n_parallel: 4           # Concurrent slots
    ctx_size: 4096          # Context window
    n_gpu_layers: all       # GPU layers offload: number or "all"
    batch: 2048             # Prefill batch size
    ubatch: 512             # Micro-batch size
    cache_key: q8_0         # KV cache key type
    cache_val: turbo4       # KV cache value type
```

**Auto-skip:** Models with `vllm_tp > cluster.gpu_count` are automatically skipped by `bench-models.sh`.

### Model Configuration

```yaml
base_dir: /workspace/models    # Root directory for all models

models:
  - name: qwen32b-awq          # Short name (used for --only filtering)
    repo_id: Qwen/Qwen2.5-32B-Instruct-AWQ
    local_dir: hf/qwen2.5-32b-awq   # Relative to base_dir
    format: hf                  # hf (full repo) or gguf (selective)
    enabled: true               # Set true to download/benchmark
    phase: p3                   # p0, p1, p2, p3, embedding
    backend: vllm               # vllm, llamacpp, sglang
    proxy_name: qwen32b-vllm    # Model name for LiteLLM proxy
    endpoint: true              # Route through proxy
    # Per-model vLLM overrides (take precedence over cluster defaults)
    vllm_tp: 1
    vllm_gpu_mem: "0.87"
    vllm_quant: awq
    vllm_max_seqs: 256
```

**Fields:**

| Field | Required | Description |
|-------|----------|-------------|
| `name` | Yes | Unique identifier for `--only` filtering |
| `repo_id` | Yes | HuggingFace repository ID |
| `local_dir` | Yes | Destination path (relative to `base_dir`) |
| `format` | No | `hf` (full repo) or `gguf` (specific files). Default: `hf` |
| `include` | No | Glob pattern for GGUF files |
| `exclude` | No | Glob pattern to skip files |
| `enabled` | No | `true` to download/benchmark. Default: `true` |
| `phase` | No | Benchmark phase: p0, p1, p2, p3, embedding |
| `backend` | No | Server backend: vllm, llamacpp, sglang |
| `proxy_name` | No | Model name for LiteLLM proxy routing |
| `endpoint` | No | `true` to route through proxy |
| `vllm_tp` | No | Tensor parallel size override |
| `vllm_gpu_mem` | No | GPU memory utilization override |
| `vllm_quant` | No | Quantization override (awq, gptq, none) |
| `vllm_max_seqs` | No | Max concurrent sequences override |

### Port Configuration

```yaml
ports:
  vllm: 8000
  llamacpp: 8001
  sglang: 8002
  embedding: 8003
  proxy: 4000
```

## litellm_config.yaml

Auto-generated by `scripts/gen-litellm-config.py` from `configs/models.yaml`.

**Sections:**

- `model_list` ‚Äî LLM backends (vLLM, llama.cpp, SGLang) + embedding model
- `router_settings` ‚Äî Load balancing strategy (least-busy)
- `litellm_settings` ‚Äî Cache (redis), timeouts, Prometheus callbacks

Regenerate after changing model configs:

```bash
uv run python scripts/gen-litellm-config.py
```

## .env

Environment variables for individual scripts. Copy the template:

```bash
cp .env_example .env
vim .env  # Uncomment and set your values
```

All scripts source `.env` automatically if it exists (does not override existing env vars).

**Orchestrator scripts** (`bench-models.sh`, `start-all-tmux.sh`) read from `configs/models.yaml` ‚Äî `.env` is for individual scripts only.

| Variable | Default | Description |
|----------|---------|-------------|
| `VLLM_MODEL` | ‚Äî | HF model path (required for `run-vllm.sh`) |
| `VLLM_TP` | `1` | Tensor parallel size |
| `VLLM_QUANT` | `none` | Quantization (awq/gptq/none) |
| `LLAMA_MODEL` | ‚Äî | GGUF model path (required for `run-llamacpp.sh`) |
| `REDIS_HOST` | `localhost` | Redis host |

## Per-Script Environment Variables

**Orchestrator scripts** (`bench-models.sh`, `start-all-tmux.sh`) read from `configs/models.yaml` ‚Äî no env vars needed.

**Individual scripts** require env vars when called standalone:

### vLLM (`run-vllm.sh`, `bench-vllm.sh`)

| Variable | Required | Default | Description |
|----------|----------|---------|-------------|
| `VLLM_MODEL` | Yes* | ‚Äî | HF model path or local dir |
| `VLLM_TP` | No | `1` | Tensor parallel size |
| `VLLM_GPU_MEM_UTIL` | No | `0.87` | GPU memory utilization |
| `VLLM_QUANT` | No | `none` | Quantization (awq/gptq/none) |
| `VLLM_MAX_MODEL_LEN` | No | `4096` | Max context length |
| `VLLM_MAX_NUM_SEQS` | No | `64` | Max concurrent sequences |
| `VLLM_BENCH_MODEL` | Yes* | ‚Äî | Model name for benchmark |
| `VLLM_BENCH_URL` | No | `http://localhost:8000` | Server URL |

### llama.cpp (`run-llamacpp.sh`)

| Variable | Required | Default | Description |
|----------|----------|---------|-------------|
| `LLAMA_MODEL` | Yes* | ‚Äî | Path to GGUF model file |
| `LLAMA_PORT` | No | `8001` | Server port |
| `LLAMA_N_PARALLEL` | No | `4` | Concurrent slots |
| `LLAMA_CTX_SIZE` | No | `4096` | Context window |
| `LLAMA_CACHE_KEY` | No | `q8_0` | KV cache key type |
| `LLAMA_CACHE_VAL` | No | `turbo4` | KV cache value type |
| `LLAMA_FLASH_ATTN` | No | `on` | Flash Attention |

### SGLang (`run-sglang.sh`, `bench-sglang.sh`)

| Variable | Required | Default | Description |
|----------|----------|---------|-------------|
| `VLLM_MODEL` / `SGLANG_BENCH_MODEL` | Yes* | ‚Äî | Model path |
| `SGLANG_TP` | No | `1` | Tensor parallel size |
| `SGLANG_QUANT` | No | `none` | Quantization |
| `SGLANG_BENCH_URL` | No | `http://localhost:8002` | Server URL |

### llama.cpp Benchmark (`bench-llamacpp.sh`)

| Variable | Default | Description |
|----------|---------|-------------|
| `LLAMA_BENCH_URL` | `http://localhost:8001/v1` | Server URL |
| `LLAMA_RESULTS_DIR` | `./results/llamacpp` | Output directory |
| `LLAMA_CONC_LEVELS` | `1 4 8 16` | Concurrency levels |
| `LLAMA_CTX_SIZES` | `1024 4096 16384` | Context sizes for long context test |

*Required ‚Äî script exits with error if not set.*
